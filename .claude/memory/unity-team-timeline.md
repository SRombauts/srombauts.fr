Internal chronology behind the author's Unity tenure, kept here because the public CVs ([[blog-as-portfolio]]) are deliberately sanitized (no manager names). The CV/LinkedIn story is framed as **two plugin phases** (Unreal then Unity), intentionally NOT perfectly aligned with the three org changes, because the plugin-focus story is more compelling.

Three actual org stages:
1. **2022-02-14 → ~mid/late 2023**: hired into the "Plastic Integration team" (led by my manager at the time). Scope: Plastic C# monolithic backend + all Ecosystem plugins; heavy emphasis on modernizing/migrating the Unreal plugin to Unreal Engine 5.
2. **~mid-2023 → Jul 2024**: Integration team merged with the "Plastic Desktop team" (Plastic Editor package + Plastic Desktop & Gluon apps). Focus shifted toward the Unity plugin; Unreal plugin → maintenance.
3. **Aug 2024 → present**: my manager left end of Jul 2024; author joined the "Plastic / VCS Tech team" (plugin + desktop devs; backend handed to a different team). Focus on the Unity package: CI, release pipeline for Unity Release Management tooling, stricter package validation. 2026 → pivot to AI Skills/demos/sharing (see `.claude/training/`).

CV/LinkedIn two-phase dates (the published framing, distinct from the org stages above, and they overlap): **Phase 1 Unreal plugin = Feb 2022 → Dec 2024**, concluding with Unreal plugin **1.12.0 on 2024-12-04** (1.x line; the brag doc confirms "1.12.0 before closing 2024"). **Phase 2 Unity package = Jun 2023 → present**; first contribution shipped in release **2.0.5 on 2023-06-01**, and the author owned release planning from **2.5.0 through 2.12.x**.

Honesty constraint the author confirmed: he touched the backend (full stack) and the Desktop/Gluon apps only "to the side"; real focus was the **Unreal plugin first, then the Unity plugin**. Don't claim hands-on desktop-app feature ownership. See [[blog-employer-framing]], [[career-timeline-pre-unity]].
